## 一、总体要求

### 1. 毕业设计目标

本次毕业设计要求每位同学在完成基础系统开发的同时，融入至少一项智能化功能（如自然语言处理、推荐算法、情感分析、大语言模型调用等），培养将 AI 技术落地到实际场景的能力。

### 2. 基本规范

| 项目 | 要求 |
|------|------|
| **技术栈** | 前端不限（Vue/React/小程序/H5均可），后端推荐 Python (Flask/Django) 或 Java (Spring Boot)，数据库 MySQL/SQLite |
| **智能功能** | 每个选题必须包含至少一个 AI/智能模块，可调用开源模型或 API（如 Hugging Face、百度千帆、讯飞星火、DeepSeek 等） |
| **代码管理** | 使用 Git 进行版本管理，在 GitHub 或 Gitee 上建立仓库 |
| **文档交付** | 毕业论文 + 系统演示视频（3-5分钟） + 答辩 PPT |
| **进度节点** | 每两周进行一次进度汇报，建议使用飞书/腾讯文档记录周报 |

### 3. 时间安排（建议）

| 阶段 | 时间 | 任务 |
|------|------|------|
| 选题与开题 | 第1-2周 | 确定选题、撰写开题报告、完成文献调研 |
| 需求分析与设计 | 第3-4周 | 完成需求文档、系统架构设计、数据库设计 |
| 核心开发 | 第5-10周 | 完成系统核心功能与智能模块开发 |
| 测试与优化 | 第11-12周 | 系统测试、性能优化、Bug 修复 |
| 论文撰写 | 第13-15周 | 撰写毕业论文、准备答辩材料 |
| 答辩 | 第16周 | 系统演示 + 论文答辩 |

### 4. 评分要点

- **选题理解（10%）**：对课题背景和意义有清晰认知
- **系统完成度（30%）**：核心功能完整可运行
- **智能功能实现（20%）**：AI 模块有实际效果，而非仅做界面展示
- **代码质量（15%）**：代码规范、有注释、Git 提交记录清晰
- **论文质量（15%）**：结构完整、表述清晰、有对比实验或用户测试
- **答辩表现（10%）**：演示流畅、回答问题准确

### 5. 注意事项

1. **禁止抄袭**：代码和论文均需通过查重，AI 辅助写作需标注并体现个人理解
2. **鼓励创新**：在基础功能之上，鼓励加入个人创意和特色功能
3. **及时沟通**：遇到技术难题请及时联系老师，不要等到截止日期前才反馈问题
4. **数据合规**：如涉及用户数据采集，需注意隐私保护，使用模拟数据或公开数据集

### 选题 1：基于大语言模型的编程学习助手

**关键词**：LLM · 代码生成 · 教育 · 对话系统

**背景与意义**：
随着人工智能技术的快速发展，大语言模型（LLM）在代码理解与生成方面展现出了强大能力。然而，编程初学者在学习过程中常面临"不知道错在哪"、"看得懂但写不出"等困境，传统的编程教学平台（如 OJ 系统）只提供判题结果，缺乏针对性的解释和引导。本课题旨在利用 LLM 构建一个智能化编程学习助手，为初学者提供代码诊断、概念解释和个性化练习推荐等服务，辅助学生建立编程思维。

**核心功能**：

- **智能代码问答**：学生输入自然语言编程问题（如"Python 中列表和元组有什么区别？"），系统调用 LLM 生成包含文字解释和示例代码的结构化回答
- **代码错误诊断**：学生粘贴出错代码和错误信息，系统定位 Bug 位置并用通俗语言解释原因，给出修复方案
- **个性化出题引擎**：根据学生历史交互记录（提问过的知识点、答对/答错的题目），自动生成难度匹配的编程练习题
- **学习轨迹看板**：以可视化图表展示学生的提问频次、知识点覆盖率、薄弱环节分布

**智能模块**：
调用 DeepSeek / 通义千问 API 实现代码理解与生成。使用 Prompt Engineering 技术设计多轮对话模板，确保回答的教学适配性。结合简单的知识点标签体系追踪学习进度，使用 SQLite 存储用户画像数据。

**技术栈建议**：Python Flask + Vue 3 + SQLite + DeepSeek API

**参考论文方向**：AI 辅助编程教育、LLM 在教学中的应用、自适应学习系统

**建议论文结构**：
1. 绪论（编程教育痛点、LLM 技术背景）
2. 相关工作（现有编程辅助工具综述）
3. 系统设计（架构设计、Prompt 设计策略、知识点标签体系）
4. 系统实现（关键模块实现细节）
5. 实验与评估（用户测试、回答质量评估）
6. 总结与展望

### 选题 2：基于情感分析的校园心理健康互助平台

**关键词**：情感分析 · NLP · 心理健康 · 社区

**背景与意义**：
大学生心理健康问题日益受到社会关注。根据相关调查，超过30%的大学生存在不同程度的焦虑或抑郁情绪，但由于"病耻感"等原因，大部分学生不愿主动寻求专业心理咨询。匿名在线社区为学生提供了一个低门槛的情感倾诉渠道。然而，纯人工管理社区存在响应滞后、高风险帖子遗漏等问题。本课题利用 NLP 情感分析技术，实现对社区帖子的情感自动检测与风险预警，辅助管理者及时发现需要关注的用户。

**核心功能**：

- **匿名社区模块**：用户注册后分配匿名身份（如"星空旅人#2847"），可发帖倾诉、回复他人、点赞鼓励，保护隐私的同时建立社区归属感
- **情感自动分析**：系统对每条帖子/回复实时进行情感倾向分析（积极/中性/消极），结果以颜色标签展示（绿/灰/橙），同时计算情感分值（-1 到 +1）
- **高风险预警机制**：当帖子情感分值低于阈值或命中关键词（如"不想活""没有意义"等），系统自动向管理员发送预警通知，并在用户界面温和地推荐学校心理咨询中心联系方式
- **心理科普资源智能推荐**：根据帖子内容语义匹配相关心理健康科普文章（如焦虑管理、考试压力应对），以"你可能感兴趣"的方式展示
- **社区情感趋势看板**：可视化展示社区整体情感走势（按日/周/月）、高频话题词云、用户活跃度统计

**智能模块**：
使用 BERT-base-chinese 在情感分类数据集上微调，或调用百度 NLP 情感分析 API 作为替代方案。关键词预警规则结合正则匹配与模型置信度双重过滤，降低误报率。科普资源推荐使用 TF-IDF 向量余弦相似度匹配。

**技术栈建议**：Python Django + Vue 3 + MySQL + Hugging Face Transformers / 百度 NLP API

**参考论文方向**：在线心理健康社区分析、中文情感分析、社交媒体风险检测

**建议论文结构**：
1. 绪论（大学生心理健康现状、在线互助社区意义）
2. 相关工作（情感分析技术、心理健康NLP应用综述）
3. 系统设计（社区架构、情感分析模型选型、预警规则设计）
4. 系统实现（核心模块实现、模型训练/API调用）
5. 实验与评估（情感分类准确率、预警召回率、用户反馈）
6. 总结与展望

### 选题 3：AI 驱动的个人简历优化与岗位匹配系统

**关键词**：NLP · 文本匹配 · 推荐系统 · 求职

**背景与意义**：
大学生求职过程中，简历质量直接影响面试机会的获取。然而，大部分学生缺乏简历撰写经验，常出现"技能描述与岗位不匹配""项目经历表述不清"等问题。同时，面对海量招聘信息，学生难以快速筛选出与自身能力匹配的岗位。本课题利用 NLP 技术实现简历的自动解析与语义匹配，帮助学生精准定位适合的岗位并获得针对性的简历优化建议，降低求职信息差。

**核心功能**：

- **简历上传与智能解析**：支持 PDF/Word 格式简历上传，利用文本提取和命名实体识别（NER）自动提取结构化信息（姓名、教育背景、技能清单、项目经历、实习经历），提取结果可人工校正
- **岗位信息管理**：支持手动录入岗位 JD 或通过爬虫从招聘网站（如实习僧、牛客）批量采集岗位信息，建立岗位数据库
- **语义匹配度分析**：将简历和岗位 JD 分别编码为语义向量，计算余弦相似度作为匹配分，按匹配度排序推荐岗位，并以雷达图展示各维度（技能、经验、学历）的匹配情况
- **智能优化建议**：针对目标岗位，调用 LLM 生成具体的简历修改建议（如"建议在项目经历中补充使用的技术栈和量化成果"），并提供优化后的参考表述

**智能模块**：
简历解析使用 pdfplumber + 正则表达式 + SpaCy/LAC 进行实体提取。语义匹配使用 Sentence-BERT（如 `paraphrase-multilingual-MiniLM-L12-v2`）或 M3E 中文嵌入模型计算文本向量相似度。优化建议调用 DeepSeek/通义千问 API，通过精心设计的 Prompt 模板生成结构化建议。

**技术栈建议**：Python Flask + React + MySQL + Sentence-Transformers + LLM API

**参考论文方向**：简历信息提取、岗位推荐系统、文本语义匹配

**建议论文结构**：
1. 绪论（大学生求职痛点、NLP技术在招聘领域的应用）
2. 相关工作（简历解析技术、文本匹配方法综述）
3. 系统设计（系统架构、简历解析流程、匹配算法设计）
4. 系统实现（各模块实现细节、Prompt 设计）
5. 实验与评估（解析准确率、匹配效果评估、用户满意度调查）
6. 总结与展望

### 选题 4：基于知识图谱的课程推荐与学习路径规划系统

**关键词**：知识图谱 · 推荐算法 · 教育 · 可视化

**背景与意义**：
高校课程体系通常包含上百门课程，课程之间存在复杂的前置依赖和知识关联。学生在选课时往往缺乏全局视角，容易出现"先修课未修就选了高阶课程"或"选课方向分散、缺乏专业深度"等问题。本课题通过构建课程知识图谱，将课程之间的先修关系、知识点关联以图的形式可视化呈现，并结合学生已修课程和成绩表现，利用图算法智能推荐下一步最优选课方案，帮助学生规划清晰的学习路径。

**核心功能**：

- **课程知识图谱构建**：根据培养方案和教学大纲，梳理课程节点（包含课程名称、学分、所属方向、知识点标签）和边关系（先修依赖、知识关联、同方向推荐），使用 ECharts 力导向图进行交互式可视化
- **学生学业档案管理**：学生登录后录入已修课程和成绩，系统自动标记图谱中已完成节点，计算各方向的学分完成进度
- **智能选课推荐**：基于图遍历算法（拓扑排序 + 路径规划），结合学生兴趣方向和当前学业状态，推荐下学期可选课程列表并给出优先级排序，附带推荐理由（如"该课程是XX方向的核心课，且你已具备全部先修知识"）
- **学习路径可视化**：以时间线形式展示从当前状态到毕业的推荐选课路径，支持多方向（如"嵌入式方向""AI方向"）路径对比

**智能模块**：
基于 NetworkX 构建有向图建模课程依赖关系，使用拓扑排序确定课程修读顺序。推荐算法结合基于内容的过滤（课程知识点标签匹配学生兴趣）和基于图的路径优化（最短路径、最少学期完成方案）。可选：调用 LLM 为推荐结果生成自然语言解释。

**技术栈建议**：Python Flask + Vue 3 + ECharts + NetworkX + SQLite

**参考论文方向**：教育知识图谱、课程推荐系统、学习路径规划

**建议论文结构**：
1. 绪论（高校选课问题分析、知识图谱技术背景）
2. 相关工作（教育领域知识图谱、课程推荐方法综述）

### 选题 5：智能新闻摘要与观点对比平台

**关键词**：NLP · 文本摘要 · 多源信息 · 可视化

**背景与意义**：
在信息爆炸时代，同一新闻事件往往被数十家媒体同时报道，各家媒体的立场、措辞和侧重点存在差异。普通读者难以逐篇阅读，容易形成单一信息源导致的"信息茧房"。本课题开发一个智能新闻聚合平台，自动采集多个新闻源的报道，利用 NLP 技术生成精炼摘要，并通过观点提取和对比展示，帮助用户从多角度理解同一事件的全貌，培养批判性思维。

**核心功能**：

- **多源新闻采集**：通过 RSS 订阅和网页爬虫从至少 5 个主流新闻源（如新华网、澎湃、界面、央广网等）实时采集新闻，按分钟级或小时级更新
- **新闻智能聚类**：对采集到的新闻进行向量化表示，使用聚类算法将报道同一事件的不同新闻归为一组（事件簇），每个簇自动生成事件标题
- **自动摘要生成**：对每篇新闻调用 LLM 生成 100 字以内的摘要，对每个事件簇生成综合摘要（融合多源信息）
- **观点对比展示**：提取各报道的核心观点/立场（如"支持""反对""中立"），以对比卡片或表格形式并排展示，高亮不同报道间的关键差异点
- **热点趋势看板**：展示当日热点事件排行、各事件的报道数量趋势图、媒体关注度分布

**智能模块**：
新闻聚类使用 Sentence-BERT 编码新闻标题和首段文本为向量，通过 DBSCAN 或层次聚类算法进行事件归组（DBSCAN 无需预设簇数，适合新闻场景）。摘要和观点提取调用 DeepSeek/通义千问 API，通过 Prompt 工程指定输出格式（JSON 结构化观点列表）。

**技术栈建议**：Python Flask + Vue 3 + MySQL + Sentence-Transformers + DBSCAN + LLM API

**参考论文方向**：多文档摘要、新闻聚类、观点挖掘、媒体偏见分析

**建议论文结构**：
1. 绪论（信息过载问题、多源新闻分析意义）
2. 相关工作（文本聚类、自动摘要、观点挖掘技术综述）
3. 系统设计（数据采集流程、聚类方案、摘要策略、观点提取方案）
4. 系统实现（爬虫实现、聚类效果调优、Prompt 设计）
5. 实验与评估（聚类纯度、摘要质量ROUGE评分、用户可读性评估）
6. 总结与展望

### 选题 6：基于图像识别的校园植物科普小程序

**关键词**：图像识别 · 微信小程序 · 科普 · 地图

**背景与意义**：
大学校园通常拥有丰富的植物资源，但大多数学生对身边的花草树木知之甚少。传统的植物标牌覆盖有限且容易损坏。本课题开发一个基于微信小程序的校园植物科普工具，结合图像识别技术，让学生通过"拍一拍"即可了解植物的种类和特征信息。同时引入打卡集卡的游戏化设计，以增强用户参与感，将校园生态环境转化为一个生动的自然课堂。

**核心功能**：

- **拍照识花**：用户拍摄或上传植物照片，系统调用图像识别模型/API 返回 Top-3 候选植物种类及置信度，用户确认后展示详细信息
- **植物百科信息**：每种植物配有详细资料页（中文名、拉丁学名、科属、形态特征、花期果期、生态习性、药用/观赏价值等），配有高清示例图片
- **校园植物地图**：基于校园平面图，在地图上以图标标注已录入的植物位置，支持按种类筛选、点击查看详情，提供"附近有什么植物"的 LBS 功能
- **打卡收藏与图鉴系统**：用户识别并打卡植物后记录到个人图鉴，以进度条展示"已集齐 X/Y 种"，设置阶段性成就徽章（如"发现10种""集齐木本植物"等）
- **社区分享**：用户可上传自己拍摄的植物照片到社区，分享拍摄地点和心得

**智能模块**：
图像识别使用百度 AI 植物识别 API 或腾讯云图像识别 API（开箱即用，识别准确率高）。进阶方案：基于 MobileNetV2 在植物分类数据集（如 PlantCLEF / Oxford 102 Flowers）上微调轻量模型，部署为后端推理服务。

**技术栈建议**：微信小程序（WXML + WXSS + JS）+ Python Flask 后端 + MySQL + 百度AI API / PyTorch

**参考论文方向**：植物图像识别、移动端深度学习部署、游戏化设计

**建议论文结构**：
1. 绪论（校园植物科普需求、图像识别技术背景）

### 选题 13：基于生成对抗网络（GAN）的图像风格迁移

- **多任务风格迁移**：在照片↔莫奈油画、夏天↔冬天 两组数据上分别训练，对比不同迁移任务的难度差异
- **生成质量评估**：使用 FID（Fréchet Inception Distance）定量评估生成图像质量，同时进行人工主观评估（邀请同学打分）
- **交互界面搭建**：使用 Gradio/Streamlit 搭建简单的 Web 界面，用户上传照片即可体验风格转换效果

**技术栈**：PyTorch、Torchvision、Gradio/Streamlit、NumPy

**数据集**：Monet2Photo / Summer2Winter / Vangogh2Photo（CycleGAN 官方数据集）

**评估指标**：FID、人工评分（1-5分制）

### 选题 14：基于强化学习的贪吃蛇/Flappy Bird 游戏智能体

**关键词**：强化学习 · DQN · 游戏 AI · 策略优化

**背景与意义**：
强化学习（Reinforcement Learning）是机器学习的重要分支，其核心思想是智能体通过与环境的交互、观察奖励信号来学习最优策略。游戏环境是强化学习研究的经典试验场——DeepMind 的 DQN 在 Atari 游戏上的突破性工作开启了深度强化学习时代。本课题以贪吃蛇或 Flappy Bird 为实验平台，从零实现 DQN 及其改进算法，通过可视化训练过程和智能体行为，帮助学生直观理解强化学习中"状态-动作-奖励"的核心循环以及深度学习在策略逼近中的作用。

**研究内容**：

- **游戏环境搭建**：使用 Pygame 实现贪吃蛇或 Flappy Bird 游戏逻辑，封装为 Gymnasium 兼容接口（reset / step / render），定义状态空间、动作空间和奖励函数
- **状态表示设计**：对比不同状态表示方案的效果：
  - 特征向量方案：提取关键信息（蛇头方向、食物相对位置、障碍距离等）组成低维向量
  - 像素方案（进阶）：直接使用游戏画面灰度图作为输入
- **DQN 算法实现**：实现经典 DQN（经验回放 + 目标网络），理解为什么需要这两个技巧来稳定训练
- **改进算法对比**：
  - Double DQN：解决 Q 值过估计问题
  - Dueling DQN：分离状态价值和动作优势
  - 对比三种算法的学习速度和最终表现
- **训练过程可视化与分析**：绘制奖励曲线（episode reward vs episode number）、Q 值变化趋势、探索率衰减曲线，录制不同训练阶段的智能体行为视频（初期随机→中期学会基本操作→后期稳定高分）
- **超参数敏感性实验**：分析学习率、折扣因子γ、经验回放容量、目标网络更新频率、ε衰减策略等超参数对训练效果的影响

**技术栈**：PyTorch、Pygame、Gymnasium、Matplotlib、imageio（用于录制GIF/视频）

**数据集**：无需外部数据集，通过游戏环境自动生成训练数据

**评估指标**：平均 Episode Reward、存活步数、学习收敛速度

**建议论文结构**：
1. 绪论（强化学习背景、深度强化学习发展里程碑）
2. 相关工作（Q-Learning 原理、DQN 及其变体综述）
3. 环境设计（游戏逻辑、状态/动作/奖励定义）
4. 算法设计（DQN / Double DQN / Dueling DQN 原理与实现）
5. 实验设置（超参数配置、训练流程、评估方案）
6. 实验结果与分析（算法对比、超参数实验、训练过程可视化、行为分析）
7. 总结与展望

### 选题 15：基于对比学习的图像相似度检索

**关键词**：对比学习 · 图像检索 · 特征提取 · 度量学习

**背景与意义**：
"以图搜图"是计算机视觉领域的重要应用，广泛用于电商商品搜索、版权保护、医学影像检索等场景。其核心挑战在于如何学习高质量的图像特征表示，使得相似图像在特征空间中距离更近、不同图像距离更远。对比学习（Contrastive Learning）是近年来自监督/半监督表示学习的主流范式，通过构造正负样本对来训练特征提取器，在标注数据有限的场景下尤为有效。本课题实现 SimCLR 等对比学习方法训练图像特征提取网络，并构建高效的以图搜图系统。

**研究内容**：

- **对比学习框架实现**：
  - SimCLR：同一图像经两种不同数据增强生成正样本对，其余图像为负样本，使用 NT-Xent（归一化温度交叉熵）损失训练
  - Supervised Contrastive Learning（进阶）：利用标签信息，同类样本为正对
  - Triplet Loss（对比方案）：锚点-正样本-负样本三元组训练
- **特征提取网络**：基于 ResNet-18（去掉最后的全连接层）构建特征提取器，添加投影头（MLP），输出 128 维特征向量
- **数据增强策略研究**：系统对比不同数据增强组合（随机裁剪、水平翻转、颜色抖动、高斯模糊、灰度化）对特征质量的影响，寻找最优增强策略
- **高效检索系统搭建**：使用 FAISS 库对图库中所有图像提取特征并建立向量索引（IVF / HNSW），实现毫秒级近邻检索
- **检索效果评估**：使用标准检索评估指标（Precision@K、Recall@K、mAP）在测试集上定量评估，对比不同对比学习方法和数据增强策略的效果
- **交互演示界面**：使用 Gradio 搭建 Web 界面，用户上传查询图片，系统返回最相似的 Top-10 图片及相似度分数

**技术栈**：PyTorch、Torchvision、FAISS、Gradio、Matplotlib

**数据集**：CIFAR-10（6万张，10类，适合快速实验）/ STL-10 / Caltech-256（更大规模）

**评估指标**：Precision@K（K=1,5,10）、Recall@K、mAP、特征空间 t-SNE 可视化

**建议论文结构**：
1. 绪论（图像检索应用背景、表示学习与对比学习意义）
2. 相关工作（传统图像检索、度量学习、对比学习方法综述）
3. 方法设计（SimCLR/Triplet Loss 原理、网络架构、数据增强策略）
4. 检索系统设计（特征提取 → 向量索引 → 近邻搜索流程）
5. 实验设置（数据集、训练配置、评估指标、对比方案）
6. 实验结果与分析（方法对比、增强策略消融、t-SNE 特征可视化、检索案例展示）
7. 总结与展望
